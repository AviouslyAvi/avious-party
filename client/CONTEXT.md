# client/ — browser-side workspace (Layer 2)

Two delivery vehicles for the same `shared/` engine. v1 = Tampermonkey userscript (shipping). v2 = Chrome MV3 extension (shipping in test, see `docs/HANDOFF.md`).

## Load

### v1: `userscript/`
- `main.ts` — top-frame entry. `@match *://*/*`. Locates `<video>`, mounts the floating panel, opens the WebSocket, wires `shared/sync.ts` to the live `<video>`. Boot is gated by the `#party=` URL hash + the `master_enabled` setting.
- `iframe-bridge.ts` — runs inside cross-origin player iframes. Hooks the iframe's `<video>`, postMessages events to the top frame, receives commands back.
- `ui/panel.ts` — sidebar UI: header, FFA + key controls, participants, chat, composer, emoji picker, settings drawer, follow-video toast, notification-permission card, sun-arc pull-out arrow.
- `banner.txt` — Tampermonkey `// ==UserScript==` header. `build.mjs` prepends it.

### v2: `extension/`
- `manifest.json` — MV3 manifest. `all_frames: true`, `match_about_blank: true` so the content script reaches cross-origin player iframes. `permissions: storage, notifications, tabs`. `web_accessible_resources` exposes bundled fonts + stylesheets to the host page so `chrome.runtime.getURL` resolves at runtime.
- `content.ts` — re-exports `../userscript/main`. The content script + iframe-bridge live in `userscript/` and are packaged for the extension runtime.
- `background.ts` — service worker. Fans master-kill changes out to all tabs.
- `popup/popup.html` + `popup.ts` — toolbar action. Three actions: start party on this page, copy invite link, master kill switch.
- `lib/settings.ts` — typed `chrome.storage.sync` wrapper with localStorage fallback for the userscript build. Owns the AVIOUS palette tokens.
- `lib/notify.ts` — sound + browser-push helpers. Sound is a generated 22kHz WAV embedded as a data URL by `build.mjs`.
- `lib/emoji.ts` — curated ~80-emoji set, 5 categories.
- `lib/video-watcher.ts` — site-aware "host changed video" detection (YouTube `yt-navigate-finish`, Netflix path polling, generic `<video src>` mutation + SPA `pushState`/`popstate`). Same-host-only by construction.
- `styles/chat.css` + `styles/popup.css` — brand-themed stylesheets. `chat.css` is imported as a string into `panel.ts` via esbuild's text loader, then injected into the host page as a `<style>` tag.
- `assets/fonts/` — bundled New Astro + Mosvita Sans (regular / medium / bold). Loaded via `chrome.runtime.getURL` at runtime when in the extension; falls back to system-ui in the userscript build.
- `_sound.generated.ts` — auto-generated by `build.mjs`. Do not edit by hand.

## Activation model

The content script loads on every tab (manifest is still `<all_urls>` because MV3 can't dynamically opt-out per-tab without on-demand registration), but boots **dormant**: no UI, no WebSocket, no observers.

Wake conditions, evaluated on init / on `hashchange` / on storage change / on popup runtime message:
1. URL hash contains `party=<id>` (the room signature emitted by `landing/app.js` and by the popup's "Start watch party" action), AND
2. `settings.master_enabled` is `true`.

The master kill switch in the toolbar popup short-circuits (1) and unmounts the session on every tab when flipped off.

## Settings

Single `chrome.storage.sync` key: `watchparty_settings_v1`. See `client/extension/lib/settings.ts` for the shape. Live-applied via `chrome.storage.onChanged`.

## Same-domain video follow

When the room admin navigates to a new same-domain video (URL change with a new video id, or `<source src>` swap), the client emits `{ type: "videoChange", url, ts }` over the existing relay. Receivers either auto-navigate (`auto_follow_video=true`, default) or surface the follow toast with a "Go" button. Cross-domain `videoChange` messages are dropped on both sender and receiver — a single system message lands in chat instead.

Protocol delta: `shared/protocol.ts` gains `VideoChangeMsg`, additive at `PROTOCOL_VERSION = 1`. Older peers ignore unknown types.

## Skip

`../node_modules/`, `../dist/`, `../shared/` (only import from it via the engine API).

## Pipeline rules

1. **Never** reach into `shared/` and add a DOM dep. Wrap it in the client adapter instead.
2. Video detection is fragile. Use a `MutationObserver` on `document` (top) and on the iframe's `document` to catch source-switch / episode-change remounts.
3. Two video adapters exist (top-frame and iframe-bridge). They share the same `VideoAdapter` interface from `shared/`.
4. No raw relay URLs in source. Read `WS_URL` from build env. See `../build.mjs`.

## Known v1 fixes already landed (don't regress)

- Capture room link at **click time**, not at boot — SPA paths (Cineby etc.) change after load.
- Chat input swallows keyboard shortcuts (spacebar, arrows) so they don't leak to the page's player.
- Drift-check before seek on remote play/pause — prevents needless buffering spinner.

## Build / release

```bash
node build.mjs                                  # default target (both)
TARGET=user node build.mjs                      # → dist/avious-party.user.js
TARGET=ext node build.mjs                       # → dist/extension/
WS_URL=wss://... TARGET=ext node build.mjs      # prod extension build
LANDING_URL=https://watch-party.pages.dev TARGET=ext node build.mjs   # popup footer link override
```

Userscript releases: paste `dist/avious-party.user.js` into Tampermonkey's editor, or host the raw `.user.js` on GitHub.
Extension releases: zip `dist/extension/`, share for "Load unpacked".
